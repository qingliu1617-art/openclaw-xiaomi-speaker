# 龙虾控制小爱音箱 🦐

> 当手机无法触达家人时，让音箱成为家庭的语音纽带

---

## 💔 一个被忽视的痛点

### 场景：你在家族群发了消息，但没人回

**"@所有人 下午3点记得去接孩子"**

结果：
- 👶 **孩子**：没有手机，完全看不到消息
- 👴 **爷爷**：手机在充电，屏幕一直黑着
- 👵 **奶奶**：老花眼，懒得翻微信
- 👨‍💼 **爸爸**：开会中，手机静音

**消息发了，等于没发。**

---

## 🔊 为什么不能依赖手机？

| 人群 | 问题 | 结果 |
|------|------|------|
| **孩子** | 没有手机 / 家长不让用 | 收不到任何通知 |
| **老人** | 不习惯看手机 / 忘记带 / 不会用 | 消息石沉大海 |
| **忙碌的成年人** | 开会、开车、手机静音 | 错过重要提醒 |
| **全家人** | 群消息太多，被淹没 | 关键信息被忽略 |

**手机是私人的，音箱是全家的。**

## ✅ 解决方案：音箱语音播报

当文字消息无法触达时，**让小爱音箱直接开口说话**。

### 你能做什么

**场景 1：提醒孩子**
```
你（在公司）→ OpenClaw → 小爱音箱：
"哥哥，4点了，该去上钢琴课了"

结果：孩子听到了，放下游戏，准备出门
```

**场景 2：提醒老人**
```
你（在出差）→ OpenClaw → 小爱音箱：
"妈，该吃降压药了，别忘了喝水"

结果：老人听到了，按时吃药
```

**场景 3：家庭协调**
```
你（在加班）→ OpenClaw → 小爱音箱：
"全家注意，今晚7点有客人来吃饭"

结果：所有人都听到了，提前准备
```

## 🎯 核心能力

### 1. 远程语音播报（突破手机限制）

无论你在哪里，只要一句话，家里的小爱就能：
- 📢 **大声喊人** - 比打电话还直接
- ⏰ **定时提醒** - 到点自动播报
- 🎵 **播放内容** - 控制音量、切歌

### 2. 自动化守护（无需人工值守）

设置一次，每天自动执行：

```bash
# 工作日上午 8:30，提醒孩子上网课
cron: 30 8 * * 1-5
播报: "哥哥，该进Zoom教室了"

# 每天下午 2:30，提醒写作业
cron: 30 14 * * *
播报: "该写今天的数学作业了"

# 每天晚上 9:30，提醒全家
cron: 30 21 * * *
播报: "该准备睡觉啦，明天还要早起"
```

### 3. 智能设备匹配（无需记忆复杂ID）

不用记 `did: 936030139`，直接说：
- "小爱" → 找到「小爱音箱Pro」
- "客厅音箱" → 匹配客厅设备
- "卧室音响" → 智能识别

## 🏠 典型使用场景

### 场景 A：双职工 + 老人带娃

**痛点**：
- 夫妻俩上班，老人在家带孩子
- 老人不会用微信，孩子没有手机
- 重要事项（吃药、接孩子、关煤气）经常遗忘

**解决**：
- 远程语音提醒老人："妈，3点去接妹妹"
- 定时提醒吃药："该吃降压药了"
- 安全提醒："检查一下煤气关了没"

### 场景 B：孩子独自在家

**痛点**：
- 孩子上网课，没有手机
- 到点不知道上课、不知道休息
- 家长在公司干着急

**解决**：
- 课前5分钟："语文课要开始了，准备进教室"
- 课间提醒："休息10分钟，保护眼睛"
- 作业时间："该写作文了，别玩游戏"

### 场景 C：跨代家庭沟通

**痛点**：
- 三代同堂，作息不同
- 消息发群里，老人看不到
- 孩子玩手机，消息被游戏通知淹没

**解决**：
- 统一语音入口，全家都听得见
- 重要事项音箱播报，不依赖任何个人设备
- 定时任务自动化，减少反复提醒的疲惫

## 🚀 快速开始

### 安装

```bash
pip3 install mijiaAPI
```

### 登录

```bash
python3 -m mijiaAPI --list_homes
# 扫码登录你的小米账号
```

### 使用

```bash
# 查看设备
python3 xiaomi-speaker.py list

# 语音播报
python3 xiaomi-speaker.py say 小爱 "该吃饭啦"

# 设置定时提醒
openclaw cron add --cron "30 12 * * *" \
  --message "让小爱播报：该吃午饭了"
```
